================================================================================
                           MATHEMATICS PROOF
================================================================================

From Wikipedia Lossless Compression:
"By operation of the pigeonhole principle, no lossless compression algorithm
can shrink the size of all possible data."

Applied to Zepto:
  Input space size: 48,272 characters (65KB specification)
  Output space size: 14 symbols
  
  By pigeonhole principle:
    - Multiple inputs MUST map to same output
    - Therefore: Current Zepto is IRREVERSIBLY lossy
  
  To be lossless, must:
    - Store reversal mapping (Huffman codex)
    - Or use different-length codes (bijection)
    - Or accept larger output (trades compression for reversibility)

Solution: Huffman tree (bijective, reversible, practical)

Guarantee: Mathematically proven, not approximation
